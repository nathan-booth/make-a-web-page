Lesson 1: The Web and HTML

The Web

The World Wide Web is a collection of connected HTML documents. HTTP stands for hypertext transfer protocol and it defines the rules for transferring files among computers, servers, and software (like browsers) on the web.

HTML

HTML stands for hypertext markup language; it is the code behind HTML documents that tells computers and browsers what information to display. An element of HTML is composed of an opening tag, content, and a closing tag, except in the case of void tags which do not require a closing tag. Elements are either inline or block; the latter forms a box around the content inside. Tags are often assigned attributes which change the functionality of an element, such as color or creating a link.

Last Words

In general, don't worry about memorizing bits of code; anything you haven't memorized by common use is easily found with a Google search or common resources, like w3schools or Stack Overflow.

Lesson 2: Structure and Tools

Document Object Model (DOM)

The DOM refers to the tree-like or nested structure of HTML documents. A web page can be viewed as a series of nested boxes in which content resides. Properly structured code uses indentation to reflect the organization of the boxes.

Tools

Chrome and other browsers have developer tools that can be used to inspect the code behind a web page.

Text editors like Sublime Text are used to write code because they make life for a programmer a bit easier by automatically completing bits of code, identifying syntax errors, and more.

Lesson 3: CSS and Philosophy

CSS

CSS stands for cascading style sheets and is the code that works in conjunction with HTML to affect the look or style of HTML documents. (Side note: CSS is usually used as a singular noun.) This is accomplished by using classes in HTML that CSS can use to affect everything of the same class plus the child elements of that class (see: inheritance).

On Repetition

Code is used to describe and execute patterns; in programs, it automates repetitious tasks. For static objects, like many documents, programmers still like to avoid repetition in order to maintain consistent code and avoid errors. In the context of HTML and CSS, using classes to make style changes means that coders need only write a code for a class and not rewrite code each time a particular effect is needed. This reduces the chance for minor errors and simplifies the debugging process.
